# AGENTS.md

## 대화와 작업 방식

- 사용자는 한국어 존대말로 응대한다.
- 파일 삭제가 필요할 때는 `rm` 대신 항상 `trash`를 사용한다. `rm`을 폴백으로 쓰지 않는다.
- 작업 전후로 `plan.md`를 확인하고, 진행 단계와 결정 사항을 기록한다.
- 중요한 판단 기준이 필요한 경우 `constitution.md`를 먼저 확인한다.
- 변경 후에는 review agent 기준으로 자체 점검한다.

## 계획 기록

- 모든 프로젝트 작업은 `plan.md`를 중심 기록으로 유지한다.
- `plan.md`에는 프로젝트의 최초 의도, 현재 상태, 선택한 방법, 진행 시각, 결정 사항, 변경 이유를 남긴다.
- 계획은 바뀔 수 있지만, 최초 기획 의도는 지켜야 한다.

## 판단 기준

- 중요한 제품/운영 판단 기준은 `constitution.md`에 정리한다.
- 판단이 애매하면 `docs/agents/constitution-agent.md`의 절차에 따라 기준을 확인한 뒤 진행한다.

## 리뷰 기준

- 새 변경이 생기면 `docs/agents/review-agent.md`의 체크리스트로 검토한다.
- 리뷰는 코드 문제, 의사결정 근거, 사용자 확인 필요 여부, `plan.md` 업데이트 여부를 함께 본다.
