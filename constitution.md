# constitution

## 목적

이 문서는 `kor-companies` 프로젝트에서 중요한 판단을 내릴 때 기준점으로 사용한다. 애매한 결정은 기능을 많이 추가하는 방향보다 운영 안정성, 재현 가능성, 사용자의 최초 기획 의도를 지키는 방향으로 판단한다.

## 핵심 원칙

1. 안정성이 발견 범위보다 우선이다.
2. 정기 운영은 GitHub Actions를 기준으로 유지한다.
3. 로컬 실행은 개발과 점검 목적을 기본값으로 둔다.
4. 상태 파일과 리포트는 운영 상태의 일부로 취급한다.
5. 한국 언론 원문, PR 배포망, 품질 낮은 재게시 소스는 가능한 한 제외한다.
6. Google News는 기본 수집원이 아니라 보조 수집원으로 다룬다.
7. 새로운 외부 의존성은 명확한 운영 이득이 있을 때만 추가한다.
8. Secrets, 토큰, API 키는 코드와 리포트에 기록하지 않는다.
9. 중복 발송 위험이 있는 변경은 보수적으로 처리한다.
10. 사용자의 확인 없이 프로젝트의 근본 운영 방식은 바꾸지 않는다.

## 의사결정 기준

- 소스 추가: 신뢰도, RSS 안정성, 국가 우선순위, 실패 시 영향도를 함께 본다.
- 매칭 로직 변경: false positive 감소를 우선하되, 주요 기업 관련 핵심 기사 누락 가능성도 테스트한다.
- 리포트 형식 변경: 텔레그램 가독성과 GitHub 기록의 재현성을 함께 유지한다.
- 상태 관리 변경: 기존 `data/state/state.json`과 호환되게 한다.
- 스케줄 변경: 한국 시간 기준 08:00, 18:00 운영 목적을 훼손하지 않는다.
- 번역 변경: 기사 전체 번역보다 제목과 회사 관련 문맥 번역을 우선해 비용과 노이즈를 줄인다.

## 사용자 확인이 필요한 경우

- 정기 실행 주체를 GitHub Actions가 아닌 다른 시스템으로 바꾸는 경우
- Telegram 또는 Google Translate 외의 새 유료 서비스를 추가하는 경우
- 상태 파일을 초기화하거나 과거 리포트를 대량 정리하는 경우
- 감시 대상 기업 또는 국가 우선순위를 크게 바꾸는 경우
- Google News를 기본 수집 경로로 승격하는 경우
