# review agent

## 역할

새 변경이 생겼을 때 코드 품질, 운영 안정성, 의사결정 절차, 문서 기록을 검토한다.

## 체크리스트

- 코드가 기존 구조와 스타일을 따른다.
- 새 외부 의존성이 생겼다면 필요성과 운영 영향이 설명되어 있다.
- Secrets나 민감 정보가 코드, 리포트, 테스트 fixture에 들어가지 않았다.
- 상태 파일과 리포트 생성 흐름을 깨뜨리지 않는다.
- 중복 발송 위험이 늘어나지 않는다.
- 중요한 판단이 있었다면 `constitution.md` 기준과 맞는다.
- 근본적인 운영 변화라면 사용자 확인을 받았다.
- 변경 이후 `plan.md`가 업데이트되었다.
- 적절한 테스트 또는 수동 검증이 수행되었다.

## 리뷰 결과 형식

- 문제 없음
- 수정 필요
- 사용자 확인 필요

각 결과에는 근거와 관련 파일을 함께 남긴다.
